# Open PR Skill

Creates a feature branch from main and opens a draft PR on GitHub. Expects: feature summary,
plan overview, plan type (feat/fix/refactor/chore/perf).

## Step 1: Derive Branch Name

```
<type>/<kebab-case-summary>
```

- `type` is the plan type (feat, fix, refactor, chore, perf)
- Summary is 3-5 words, kebab-case, derived from the feature description
- Max 50 characters total, no special characters beyond hyphens
- Examples: `feat/add-streak-tracking`, `fix/timer-race-condition`

## Step 2: Create Branch

```bash
git checkout -b <branch-name>
```

If the branch already exists, ask the user whether to reuse it or create a new one with a
numeric suffix (e.g., `feat/add-streak-tracking-2`).

## Step 3: Push Branch and Open Draft PR

```bash
git push -u origin <branch-name>
gh pr create --draft --title "<PR title>" --body "<PR body>"
```

**PR title:** `<type>(<scope>): <imperative summary>` — max 72 characters, conventional-commit format.

**PR body template:**

```markdown
## Summary

<1-3 sentences describing what this PR implements and why>

## Plan

<High-level description of the architect's approach>

### Tasks

**Wave 1:**
- [ ] Task 1.1: <name> (<model>)
- [ ] Task 1.2: <name> (<model>)

**Wave 2:**
- [ ] Task 2.1: <name> (<model>)

### Key Decisions

<2-5 bullet points of architectural decisions made by the architect>

## Verification

- [ ] All tasks implemented and committed
- [ ] Code review passed (code-reviewer)
- [ ] Tests passing with >= 90% coverage
- [ ] ORCHESTRATOR.md updated (if architecture changed)

## Coverage

_Will be updated by the orchestration pipeline after tests run._

---

Generated by the orchestration pipeline.
```

## Step 4: Return PR Info

Output so the orchestrator can track it:

```
PR_BRANCH: <branch-name>
PR_NUMBER: <number>
PR_URL: <url>
```

**Error notes:** If `gh` is not installed, print "Install GitHub CLI: brew install gh" and stop.
If not authenticated, print "Run: gh auth login" and stop. If branch conflicts, ask user.
